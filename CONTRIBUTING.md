# Contributing Guidelines

For those who would like to contribute to the WxData Project, please take note of the following guidelines. 

In order for your pull request to be accepted, you must comply with the following guidelines:

1) Only publicly available shapefiles may be hosted in this repository.
2) Absolutely no shapefiles generated by private companies are allowed to be hosted in this github repository - even if the shapefiles are publicly available. 
3) You are not allowed to use packages that are not available on BOTH Anaconda and PYPI (pip) 
    - This project must be available on PYPI in addition to Anaconda to maximize access for use.
4) The use of setup.py files is forbidden due to security vulerabilities associated with the use of setup.py. We use pyproject.toml for building our recipe.
5) If you are adding a new package dependency to shapeography, please cite it:
     - [dependency citations](https://github.com/edrewitz/shapeography/tree/main?tab=readme-ov-file#citations)
6) Be willing to accept feedback and constructive criticism.
7) Be respectful.
8) Have fun!
